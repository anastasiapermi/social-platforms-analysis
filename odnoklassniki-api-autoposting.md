# Одноклассники (OK) API — автопостинг

Официальная документация: https://apiok.ru/dev
Основные методы: `mediatopic.post` (публикация), `photosV2.*` (загрузка фото), `video.upload` (загрузка видео).

## 1. Summary

OK API позволяет публиковать в ленту пользователя, группу или в статус через единый метод `mediatopic.post`. Контент — JSON-объект `attachment` с массивом `media` различных типов (text/photo/movie/music/poll/link/app). Постинг в группу требует `GROUP_CONTENT`, медиа — `PHOTO_CONTENT`/`VIDEO_CONTENT`. **Особенность авторизации:** каждый запрос подписывается MD5 (нестандартная схема, требует подписи на нашей стороне).

**Сложность интеграции: medium-high** из-за подписи запросов, верификации сервисов и многошагового upload медиа.

## 2. Что нужно для запуска

1. Зарегистрированное OK-приложение (https://ok.ru/devaccess).
2. `application_key` (публичный) и `application_secret_key` (секретный).
3. Верификация сервисов ОК (модерация приложения) — для production.
4. Permissions/scope: `VALUABLE_ACCESS`, `GROUP_CONTENT` (для групп), `PHOTO_CONTENT`, `VIDEO_CONTENT`.
5. Access token пользователя через OAuth (или сессия для встроенных приложений).
6. ID целевой группы (`gid`) или пользователя (`uid`).

## 3. Авторизация

Документация общая: https://apiok.ru/dev/methods/

- База API: `https://api.ok.ru/fb.do?method=<method.name>&...` или `https://api.ok.ru/api/<group>/<name>?...`.
- Формат ответа: только JSON.

### OAuth

- Standalone и web-приложения проходят OAuth flow для получения `access_token`.
- Для внутренних встроенных приложений (виджеты, мини-приложения) — `session_key` без OAuth.
- Тип доступа определяет, какие permissions может запросить приложение.

## 4. Права / scopes (permissions)

Для автопостинга нужны:

| Permission       | Назначение                              |
|------------------|-----------------------------------------|
| `VALUABLE_ACCESS`| базовое расширенное API (требуется почти всегда) |
| `GROUP_CONTENT`  | публикация в группу                     |
| `PHOTO_CONTENT`  | загрузка/прикрепление фото              |
| `VIDEO_CONTENT`  | загрузка/прикрепление видео             |
| `SET_STATUS`     | установка статуса пользователя          |
| `PUBLISH_NOTE`   | публикация заметок                      |

Permissions запрашиваются при OAuth-консенте пользователя и должны быть одобрены при верификации приложения.

### Активация приложения

Для production использования приложение проходит **модерацию ОК** в кабинете разработчика (https://ok.ru/devaccess). До активации permissions работают только в режиме разработки.

> **Важно:** страница «Верификация сервисов ОК» в документации (https://apiok.ru/dev/verification) **не про App Review** — это про верификацию входящих запросов от сервисов OK к нашему серверу (web grabber, BOT API, payment validation) по IP/User-Agent. Не путать.

## 5. Адресация

`mediatopic.post` принимает:

- `uid` — ID пользователя (от имени которого пост).
- `gid` — ID группы (для постинга в группу; вместе с `type=GROUP_THEME` или `GROUP_THEME_PROMO`).
- `type` — тип медиатопика:

| Тип                  | Назначение                                                         |
|----------------------|--------------------------------------------------------------------|
| `USER`               | пост на страницу пользователя (видят его друзья в ленте)           |
| `USER_STATUS`        | установить в статус (требует `SET_STATUS`)                         |
| `USER_NOTE`          | заметка пользователя (требует `PUBLISH_NOTE`)                      |
| `GROUP_THEME`        | пост в группу                                                      |
| `GROUP_THEME_PROMO`  | промо-пост в группу                                                |
| `GROUP_SUGGESTED`    | предложенная тема (модерируется администрацией группы)             |

## 6. Постинг текста

Документация: https://apiok.ru/dev/methods/rest/mediatopic/mediatopic.post

Текст передаётся как элемент `media` типа `text` внутри `attachment` (структура: `{"media": [{"type": "text", "text": "..."}]}`).

Лимит длины текста явно не задокументирован; на практике — несколько тысяч символов. Форматирование (Markdown/HTML) не поддерживается.

## 7. Постинг медиа

`attachment.media` — массив объектов: `text` (текст), `photo` (массив фото-токенов), `movie` (видео-токены), `link` (одна ссылка на пост), `poll` (голосование). Прочие типы — `music`, `app`, `app-ref`, `topic`, `movie-reshare` — для нашего автопостинга не релевантны.

### Загрузка фото (через `photosV2`)

Документация: https://apiok.ru/dev/methods/rest/photosV2/

Многошаговый процесс:
1. `photosV2.getUploadUrl` — получить URL для загрузки.
2. POST фото на полученный URL → возвращается `photo_id` и токен.
3. `photosV2.commit` — подтвердить загрузку, получить итоговый `token`.
4. Использовать token в `mediatopic.post`: `{"type":"photo","list":[{"id":"<token>"}]}`.

Ограничения (документация: https://apiok.ru/dev/methods/rest/photosV2/photosV2.commit):

- Все фото должны принадлежать текущему пользователю.
- Все фото в одном альбоме.
- При посте в группу — фото не должны быть привязаны к альбому, и `photosV2.commit` для них не вызывается.
- `photoId` (для решары) — фото должно быть существующим и открытым.

### Загрузка видео

Через `video.upload` или `video.getUploadUrl` — аналогичная цепочка. После загрузки → ссылаться по ID в `media`.

## 8. Rate limits

Документация: https://apiok.ru/dev/limits

Прямая цитата из документации:

> Для практически всех REST методов отсутствуют ограничения на количество и частоту вызовов. Лимиты могут появиться в индивидуальном порядке, если вызовы приложения негативно влияют на работу API.

## 9. Особенности

- **MD5-подпись каждого запроса** — нестандартная схема. Адаптер должен подписывать каждый вызов; ошибки подписи возвращают `PARAM_SIGNATURE` (код **104**, не 101).
- **Модерация приложения** в кабинете разработчика для production. «Верификация сервисов ОК» из документации — отдельная тема, про верификацию **входящих** запросов от OK к нашему серверу, **не про App Review**.
- **Rate limits официально не публикуются** для REST-методов. Появляются индивидуально при злоупотреблении. Сигнал — ошибка `FLOOD_BLOCKED` (код 8) с `wait_millis`.
- **Многошаговый upload медиа** — `photosV2.getUploadUrl` → upload → `photosV2.commit`. Кешировать токены.
- **Native scheduled publishing** — поддерживается через параметры `publishAtMs` (Unix timestamp в миллисекундах) или `publishAt` (формат `YYYY-mm-dd hh:mi:ss` в таймзоне Europe/Moscow GMT+3). Документация: [mediatopic.post](https://apiok.ru/dev/methods/rest/mediatopic/mediatopic.post). Можно отдавать запланированные посты в OK, не держа в нашей очереди.
- **`app` блок отображается в ленте только у подтверждённых приложений** — после верификации.

## 10. Что учесть при запуске интеграции

- **Модерация приложения OK** — заложить дни (быстрее всех других платформ с ревью). Без неё работает только режим разработки.
- **UX подключения канала**: клиент-админ сообщества проходит OAuth → выдаёт нам permissions (`VALUABLE_ACCESS`, `GROUP_CONTENT`, `PHOTO_CONTENT`, `VIDEO_CONTENT`). Стандартный OAuth flow.
