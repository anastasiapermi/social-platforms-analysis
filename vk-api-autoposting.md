# VK API — автопостинг

Официальная документация: https://dev.vk.com/ru/api
Основные методы: `wall.post`, `photos.getWallUploadServer`, `photos.saveWallPhoto`, `video.save`.

## 1. Summary

VK предлагает богатый API: текст, до 10 медиа, опросы, нативное отложенное постинг (`publish_date`), карусель/сетка, гео, donut-подписки. **Главный блокер для интеграции — выдача прав:** scope `wall` и `photos` для **user-токена** выдаются «в исключительных случаях через запрос в `devsupport@corp.vk.com`». Альтернатива — **community access token**, который админ сообщества создаёт в UI самостоятельно с нужными правами.

**Сложность интеграции: medium-high** (зависит от выбранного типа токена и одобрения VK).

## 2. Что нужно для запуска

1. Зарегистрированное VK-приложение (https://dev.vk.com/admin).
2. Один из двух типов токенов с правами `wall` и `photos`:
   - **Community access token** — создаётся админом сообщества в UI, проще получить.
   - **User access token** — через VK ID OAuth, scope `wall`/`photos` требует email-одобрения от поддержки.
3. ID сообщества, в которое постим (`owner_id` с минусом).

## 3. Авторизация

Документация: https://dev.vk.com/ru/api/access-token/getting-started, https://dev.vk.com/ru/api/api-requests

- База API: **`https://api.vk.ru/method/<methodName>`** (домен `api.vk.ru`, не `.com`).
- Токен передаётся в заголовке: `Authorization: Bearer <KEY>`.
- В POST `Content-Type`: `application/x-www-form-urlencoded` или `multipart/form-data`. **JSON не поддерживается.**
- Обязательный query-параметр `v` — версия API (актуальная **5.131**).

### Три типа токенов

| Тип            | От чьего имени      | Срок действия | Где использовать |
|----------------|---------------------|---------------|------------------|
| Сервисный      | приложение          | без срока     | только сервер    |
| Пользовательский | пользователь      | **1 час**     | клиент / сервер  |
| Сообщества     | сообщество / паблик | без срока     | клиент / сервер  |

### Устаревшие способы (отключены)

- **Authorization Code Flow** для user-токена — отключён **25 июня 2024**.
- **Implicit Flow** для user-токена — отключён **25 июня 2024**.
- **Client Credentials Flow** для сервисного — отключён **17 октября 2022**.

Новые токены получают через **VK ID** OAuth flow (новый).

### Получение community-токена

Документация: https://dev.vk.com/ru/api/access-token/community-token/in-community-settings

Два способа:
1. **Вручную в UI сообщества**: Управление → Дополнительно → Работа с API → Ключи доступа → Создать ключ. Подтверждение через мобильное приложение или SMS.
2. **Программно через VK ID**: Authorization Code Flow (сервер) или Implicit Flow (клиент) — после `groups.get?filter=admin`.

Все созданные ключи отображаются в настройках сообщества и могут быть отозваны.

## 4. Права / scopes

Документация на каждом методе указывает требуемые права.

Для автопостинга нужны:
- `wall` — публикация на стене (требуется для `wall.post`).
- `photos` — загрузка фото (требуется для `photos.getWallUploadServer`).
- `video` — загрузка видео (требуется для `video.save`).
- `groups` — управление сообществами (для `groups.get?filter=admin` при получении токена сообщества).

### КРИТИЧНО для user-токена

Прямая цитата с документации `wall.post`:

> Для вызова метода можно использовать:
> • ключ доступа пользователя, полученный в Standalone-приложении через Implicit Flow (требуется право доступа: **wall — выдаётся в исключительных случаях через запрос в поддержку по электронной почте devsupport@corp.vk.com**)

То же для `photos.getWallUploadServer`:

> требуется право доступа: photos — выдаётся в исключительных случаях через запрос в поддержку

Для **community-токена** эти права выдаются нормально при создании ключа в UI (админ сам выбирает permissions).

## 5. Адресация

Параметр `owner_id` в `wall.post`:
- **`123456`** — стена пользователя.
- **`-123456`** — стена сообщества (минус обязателен).

Дополнительно:
- `from_group=1` — пост от имени сообщества (только если `owner_id < 0`).
- `signed=1` — подпись «опубликовано пользователем X» (только при `from_group=1`).
- `mark_as_ads=1` — отметка «Это реклама».

## 6. Постинг текста

Документация метода: https://dev.vk.com/ru/method/wall.post

Параметры:
- `message` (text) — текст. Обязателен, если нет `attachments`.
- `attachments` (string) — медиа через запятую.

Длина текста в публичной API-документации VK явно не зафиксирована (на странице [wall.post](https://dev.vk.com/ru/method/wall.post) у параметра `message` указан только тип `text`, без лимита). На стороне нашего сервиса имеет смысл задать собственный валидационный потолок.

Форматирования через parse_mode/markdown **нет** — VK не поддерживает разметку в постах. Ссылки распознаются автоматически. Упоминания: `[id123|текст]` или `[club123|текст]` (наследие старого синтаксиса).

## 7. Постинг медиа

Документация: [wall.post](https://dev.vk.com/ru/method/wall.post) (раздел «Ограничения на публикуемый контент»), [загрузка фото](https://dev.vk.com/ru/api/upload/wall-photo).

### Что можно в одном посте

- До **10 медиа** (фото + видео в любом миксе).
- До **10 музыкальных треков** ИЛИ 1 плейлист/альбом/подкаст (только с фото/видео в режиме карусели; в сетке аудио нельзя).
- **1 опрос** (длина ответов до 100 символов; не может быть единственным вложением).
- **1 файл** (документ).
- Одна внешняя ссылка (больше одной — ошибка).

### Загрузка фото — три шага

`photos.getWallUploadServer` → получаем upload URL → POST файла на этот URL → `photos.saveWallPhoto` → получаем `{owner_id}_{photo_id}`. Этот идентификатор передаётся в параметре `attachments` метода `wall.post` (формат `photo{owner_id}_{id}`, через запятую без пробелов). Видео — аналогично через [`video.save`](https://dev.vk.com/ru/method/video.save) + загрузка на полученный URL.

### Лимиты на фото

- Форматы: JPG, PNG, GIF.
- Сумма высоты и ширины ≤ **14 000 пикселей**.
- Файл ≤ **50 МБ**.
- Соотношение сторон не менее 1:20.

### Режим отображения

- `primary_attachments_mode`: по умолчанию **карусель**, `grid` — сетка.
- `photo_attachments_crop` — параметры обрезки фото. **С 4 июня 2026** нестандартные соотношения автоматически кадрируются.

### Требования к изображениям в специальных позициях

- GIF: aspect ratio 0.66:1 — 2.5:1.
- Картинка сниппета статьи: 1:1 — 1:2, ширина ≥ 550 px.
- Картинка сниппета ссылки: 1:1 — 1:2.5, ширина ≥ 500 px.

## 8. Rate limits

Документация: https://dev.vk.com/ru/api/api-requests (раздел «Частотные ограничения») и https://dev.vk.com/ru/api/errors-handling

### Частотные лимиты (официально из живой документации)

- **User-токен**: **3 запроса/сек**.
- **Community-токен**: **20 запросов/сек**.
- **Service-токен** (масштабируется по числу пользователей приложения):

| Пользователей приложения | RPS |
|---|---|
| до 10 000 | 5 |
| до 100 000 | 20 |
| до 500 000 | 40 |
| до 1 000 000 | 50 |
| больше 1 000 000 | 60 |

Превышение → код ошибки **6 «Too many requests per second»**.

### Количественные лимиты

Помимо частоты, есть лимиты на количество однотипных вызовов. **VK явно отказывается публиковать точные числа**: «по понятным причинам мы не предоставляем информацию о точных лимитах». После превышения может потребоваться captcha (см. `captcha_error`) или метод временно блокируется.

## 9. Особенности

- **Главный блокер — scope `wall` для user-токена выдаётся по email-заявке** в `devsupport@corp.vk.com`. Решение для бизнеса — community-токен (свои права назначаются админом сообщества).
- **Native scheduled publishing** через `publish_date` (Unix timestamp) — VK сам отложит публикацию. Можно не строить очередь на нашей стороне для запланированных постов.
- **Гиперссылки могут быть запрещены** — зависит от настроек сообщества и типа поста.
- **3-шаговый upload медиа** — getServer → upload → save, токен сохраняется как `{owner_id}_{media_id}`.
- **С 4 июня 2026** — авто-кадрирование фото в карусели нестандартных размеров.

## 10. Что учесть при запуске интеграции

- **Тип токена**: для нашего use-case используем **community token** (получение через UI сообщества силами клиента, без переписки с VK support).
- **UX подключения канала**: админ сообщества создаёт API-ключ в настройках своей группы (Управление → Дополнительно → Работа с API → Создать ключ) и копирует к нам. Требуется пошаговая инструкция в нашем UI.
