# MAX Bot API — автопостинг

Официальная документация: https://dev.max.ru/docs-api
Основные методы: `POST /messages`, `POST /uploads`.

## 1. Summary

Простой stateless API для постинга в личные/групповые чаты MAX. Авторизация — статический bot-токен из платформы MAX. Поддерживает текст, медиа (4 типа), несколько медиа в одном сообщении, inline-клавиатуру. Загрузка медиа — трёхшаговая (URL -> upload host -> token).

**Сложность интеграции: low-medium** (медиа сложнее из-за асинхронной обработки и `attachment.not.ready`).

## 2. Что нужно для запуска

1. Бот, зарегистрированный на платформе MAX (Чат-боты -> Интеграция).
2. Access-token бота в секрет-сторе.
3. Целевой адресат: `user_id` (личка) или `chat_id` (групповой чат).
4. Для группового чата — бот добавлен админом с правом `write`.

## 3. Авторизация

Документация: https://dev.max.ru/docs-api (раздел «Общее описание»).

- Токен берётся в UI платформы MAX (Чат-боты -> Интеграция -> Получить токен).
- База API: `https://platform-api.max.ru`.
- Передача **только в заголовке** `Authorization: <token>`. Query-параметр больше не поддерживается.
- Токен может быть отозван за нарушения.

## 4. Права / scopes

Для постинга в групповой чат бот должен быть админом с правом `write`. Назначение бота админом — **клиент делает вручную в приложении MAX** через настройки чата.

Перечень прав, которые можно выдать боту (`ChatAdmin.permissions`):

| Право                | Назначение                                |
|----------------------|-------------------------------------------|
| `write`              | **публикация сообщений** — обязательно    |
| `read_all_messages`  | чтение всех сообщений                     |
| `add_remove_members` | добавление/удаление участников            |
| `add_admins`         | назначение администраторов                |
| `change_chat_info`   | смена названия / описания / аватара       |
| `pin_message`        | закрепление сообщений                     |

Для постинга в **личку** прав не нужно — достаточно `user_id`.

## 5. Постинг текста

Документация: https://dev.max.ru/docs-api/methods/POST/messages

- Лимит текста — **4000 символов**.
- Поддерживается форматирование через **Markdown** или **HTML** (на выбор, параметр `format`).
- Можно прикреплять вложения (медиа, inline-клавиатуру), ссылки на другие сообщения, отключать уведомление подписчикам.

## 6. Постинг медиа

Документация: https://dev.max.ru/docs-api/methods/POST/uploads

В одном сообщении можно прикрепить **несколько медиа** (массив `attachments`), но каждый файл загружается отдельным запросом.

### Почему загрузка медиа — три шага

Каждое медиа в посте требует трёх шагов: (1) спросить у MAX, на какой сервер заливать; (2) загрузить байты, получить токен; (3) опубликовать пост, прикрепив этот токен.

**Что это значит для нас:** на каждый файл — 2 дополнительных HTTP-запроса плюс публикация. Между шагом 2 и 3 MAX обрабатывает файл асинхронно (см. §8). Токены можно переиспользовать — если клиент часто постит один и тот же логотип, загружаем один раз. Пост с несколькими медиа = N параллельных загрузок + одно сообщение со списком токенов.

### Ограничения

Документация: https://dev.max.ru/docs-api/methods/POST/uploads

- Максимум **4 ГБ** на файл.
- За один вызов `/uploads` — один файл.
- Срок жизни URL для загрузки не ограничен.

## 7. Rate limits

Документация: https://dev.max.ru/docs-api (раздел «Рекомендации по работе с API»).

Глобальный лимит: **30 rps на `platform-api.max.ru`**. Per-chat / per-user раздельных лимитов нет.

Платного тарифа для повышения лимитов в публичной документации нет.

## 8. Особенности

- **`attachment.not.ready`** — после загрузки медиа платформа обрабатывает файл асинхронно. Отправка сообщения сразу может вернуть `{"code":"attachment.not.ready"}`. Стратегия: бэкофф (1с -> 2с -> 4с -> ... -> 30с) или предзагрузка с кешированием токенов.
- **Двухэтапный upload** — нельзя послать медиа одним запросом. Минимум 2 HTTP-вызова + ожидание обработки.
- **`type=photo` устарел** — старые интеграции нужно мигрировать на `type=image`.
- **Длина текста 4000 символов**.
- **`app review` отсутствует** — бот сразу боевой.

## 9. Что учесть при запуске интеграции

- **UX подключения канала**: клиент сам (1) получает токен в кабинете MAX (Чат-боты → Интеграция → Получить токен) и вставляет к нам; (2) добавляет нашего бота админом с правом `write` через настройки чата. Оба шага — действия в приложении MAX, требуют пошаговой инструкции в нашем UI.
